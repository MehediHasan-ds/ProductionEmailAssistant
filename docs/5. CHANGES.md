# Changes from the Plans

Where this file and the earlier docs disagree, this file wins.

1. I dropped DeepSeek because it is not free, and I dropped direct OpenAI. The providers are now gpt oss 120B via OpenRouter and gemini 2.5 flash lite via Google AI Studio.
2. I run Gemini on the native generateContent endpoint with the x-goog-api-key header via httpx, not the OpenAI compatible layer, because the configured key type requires it.
3. I run the local Jina ONNX model directly via onnxruntime, not optimum plus torch, because optimum 2.x is incompatible with transformers 5.x. Pooling and normalization are unchanged.
4. I keep the two keys in the gitignored .env file as OPENROUTER_API_KEY and GEMINI_API_KEY, so they never enter the repo.
5. I run generation and judging in plain mode, not with response_format json mode, because OpenRouter's gpt oss 120B returns an empty body under json mode. The prompt and the JSON parser enforce structured output instead.
6. I reduced the evaluation from 12 to 10 scenarios by removing the rare multi intent case and the casual hackathon announcement, because Gemini's free tier rate limits (429) made a 12 scenario run unreliable. The reduction applies to both providers for a fair comparison. The remaining 10 cover normal, complex, failure, and malicious categories with full coverage.
7. I upgraded the system prompt from a concise style to a depth-calibrated style. The old prompt said "keep it concise" which suppressed detail. The new prompt says "match the depth to the situation" with specific tone calibration, natural transitions, and recipient-aware reasoning. Few-shot examples were upgraded to demonstrate this calibration. The judge now uses pydantic structured output with retry instead of manual JSON parsing.
