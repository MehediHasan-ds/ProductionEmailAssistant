# Edge Case Registry

Every entry follows the same form: incident, root cause, fix, and the regression check that would catch a return of the problem. Seeded from real failures hit during the build.

## EC 001. Gemini rejected with access token unsupported

- Incident: Gemini calls returned 401 with access token type unsupported on the OpenAI compatible endpoint.
- Root cause: the configured key type is not accepted as a bearer token by the OpenAI compatible layer.
- Fix: Gemini now calls the native generateContent endpoint with the x-goog-api-key header via httpx.
- Regression check: any live generate or evaluate call to the gemini provider surfaces a 401 or 400 immediately.

## EC 002. Gemini system instruction rejected with 400

- Incident: Gemini returned 400 invalid value at system instruction.
- Root cause: the native API requires a Content object, not a raw string, for system instruction.
- Fix: system instruction is sent as a Content object with a single text part.
- Regression check: covered by any generation that includes a system prompt.

## EC 003. OpenRouter returned an empty body under json mode

- Incident: with response_format json_object, gpt oss 120B returned zero characters, so parsing failed.
- Root cause: this model empties its content under json mode on OpenRouter.
- Fix: generation and judging run in plain mode and rely on the prompt plus the JSON parser.
- Regression check: a generate call returning empty content fails the response parser.

## EC 004. Embedder produced poor similarity until BOS was added

- Incident: cosine similarity looked uniform until the BOS token was verified missing.
- Root cause: the generic tokenizer class does not prepend BOS, and EuroBERT requires it at position zero for correct positional encoding.
- Fix: the embedder prepends the BOS token before inference.
- Regression check: the embedder smoke check asserts BOS presence and that related phrases outscore unrelated ones.

## EC 005. Boot depended on a machine specific path

- Incident: the embedder model directory defaulted to an absolute path on one machine.
- Root cause: a non portable default baked into configuration.
- Fix: embedder_model_dir is required and must come from the environment.
- Regression check: starting the app without it fails fast with a clear validation error.

## EC 006. Optimum was incompatible with transformers 5

- Incident: adding optimum plus torch conflicted with transformers 5.x at resolve time.
- Root cause: optimum 2.x requires transformers below 4.58.
- Fix: the ONNX model runs directly through onnxruntime with no optimum and no torch.
- Regression check: the dependency resolve and the embedder import succeed together.
