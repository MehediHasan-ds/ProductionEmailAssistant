# Implementation Plan. Stages 10 and 11 of the ASDL

This file turns the architecture into code. It is ordered into milestones, and each milestone is independently verifiable, so I stop and prove a piece works before building the next one on top of it. The pseudocode here is illustrative, not final, but it fixes the shape of each piece so the implementation is mechanical once I start.

## Ground rules for the build

- I build the smallest viable version first and add complexity only if evaluation demands it.
- Every evaluation path runs at temperature zero so results are reproducible.
- The agent never sends anything. It drafts, scores, and refines only.
- Every dependency can fail, so each one has a fallback or a graceful degradation.
- I do not move to the next milestone until the current one passes its verification step.
- I do not persist raw facts in traces or logs by default, because inputs may contain sensitive or personally identifying content.
- If the embedding engine fails at runtime, the evaluator degrades the semantic metric to lexical only rather than failing the whole request.

## Milestone 1. Foundation

I stand up the three pieces everything else depends on: configuration, the provider client, and the embedding engine.

**Configuration** centralizes provider settings, thresholds, model paths, and weights in one place.

**The provider client** is a single function that talks to OpenAI, DeepSeek, and Gemini through their shared OpenAI compatible interface.

```
def chat(messages, provider, temperature=0.0):
    cfg = PROVIDERS[provider]
    client = OpenAI(base_url=cfg.base_url, api_key=cfg.api_key)
    resp = client.chat.completions.create(
        model=cfg.model,
        messages=messages,
        temperature=temperature,
        response_format={"type": "json_object"},
    )
    return resp.choices[0].message.content
```

**The embedding engine** loads my local quantized Jina model once at startup via the optimum plus torch path on CPU, then exposes encode and cosine helpers.

```
def encode(texts):
    texts = [PREFIX + t for t in texts]
    enc = tokenizer(texts, padding=True, truncation=True, max_length=512, return_tensors="pt")
    lhs = model(**enc).last_hidden_state
    seq_len = enc["attention_mask"].sum(dim=1) - 1
    emb = lhs[torch.arange(lhs.size(0)), seq_len]
    emb = normalize(emb, p=2, dim=1)
    return emb.cpu().numpy()
```

**Verification.** I run the two build time safety checks from the architecture. I confirm the BOS token sits at the start of the tokenized output, and I run a cosine sanity check where related phrases score high and unrelated phrases score low. I also smoke test the chat function against each provider with a trivial prompt.

## Milestone 2. The evaluation dataset and rule based metrics

I build the instrument before the agent, because the agent is only as trustworthy as the instrument that grades it.

**The dataset** holds the scenarios across all five categories, each with a hand written reference email.

- Four normal cases: a follow up after a meeting, a weekly status update, a client onboarding welcome, and a casual internal announcement
- Three complex cases: an apology with a discount, an overdue payment reminder, and a diplomatic decline
- One rare case: a multi intent email that bundles two requests in one
- Two failure cases: a missing facts scenario and a contradictory facts scenario
- Two malicious cases: a prompt injection attempt and an exfiltration attempt smuggled through the inputs

**The rule based metrics** are deterministic and free of LLM cost.

```
def rule_based_metrics(email, intent, facts, tone):
    return {
        "fact_coverage": fraction_of_facts_present(email, facts),
        "tone_match": tone_lexicon_and_sentiment_score(email, tone),
        "structure": has_subject_greeting_body_closing_signature(email),
        "length": length_band_score(email),
        "readability": flesch_reading_ease(email),
        "placeholder_leak": penalty_for_brackets_or_todos(email),
        "hallucination_flag": invented_content_score(email, facts),
        "redundancy": ngram_repetition_ratio(email),
    }
```

**Verification.** I write unit tests for the rule based metrics against hand crafted inputs with known expected scores, and those tests must pass.

## Milestone 3. The generator, the judge, and the hybrid evaluator

I build the generation and scoring core.

**The generator** assembles the prompt from role playing, two few shot examples, and a chain of thought instruction that forces structured JSON.

```
def build_prompt(intent, facts, tone, feedback=None):
    system = ROLE_PLAY_PERSONA
    user = render_template(FEW_SHOT_EXAMPLES, intent, facts, tone, feedback)
    return [{"role": "system", "content": system}, {"role": "user", "content": user}]

def generate(prompt, provider):
    raw = chat(prompt, provider)
    return parse_json(raw)
```

The parse step returns the reasoning, the subject, and the body.

**The judge** is a separate call, rubric scored and blind to the reference.

```
def llm_judge(email, intent, facts, tone):
    return chat(JUDGE_PROMPT.format(email, intent, facts, tone), JUDGE_PROVIDER)
```

The judge returns tone fidelity, fact integration, professionalism, clarity and coherence, intent alignment, and overall.

**Reference metrics** combine lexical overlap and semantic similarity.

```
def reference_metrics(email, reference):
    return {
        "rouge_l": rouge_l_score(email, reference),
        "bleu": bleu_score(email, reference),
        "cosine": embedder.cosine(email, reference),
    }
```

**The evaluator** aggregates all three groups into one weighted score.

```
def evaluate(email, intent, facts, tone, reference=None):
    rule = rule_based_metrics(email, intent, facts, tone)
    judge = llm_judge(email, intent, facts, tone)
    ref = reference_metrics(email, reference) if reference else {}
    overall = weighted_sum(rule, judge, ref, WEIGHTS)
    return {"rule": rule, "judge": judge, "reference": ref, "overall": overall}
```

**Verification.** I run one full generate and evaluate pass on a single normal scenario and confirm a sensible structured email and a complete score breakdown come back.

## Milestone 4. The self refinement loop

I add the enforced review pass that the root cause demanded.

```
def run_agent(intent, facts, tone, provider, max_attempts=3, threshold=80):
    best = None
    feedback = None
    trace = []
    for attempt in range(1, max_attempts + 1):
        prompt = build_prompt(intent, facts, tone, feedback)
        draft = generate(prompt, provider)
        scores = evaluate(draft, intent, facts, tone)
        trace.append({"attempt": attempt, "draft": draft, "scores": scores})
        if best is None or scores["overall"] > best["scores"]["overall"]:
            best = {"draft": draft, "scores": scores}
        if scores["overall"] >= threshold:
            break
        feedback = build_critique(scores)
    best["trace"] = trace
    best["attempts"] = len(trace)
    return best
```

**Verification.** I run a deliberately weak first draft scenario and confirm the trace shows the score climbing across attempts, or stopping cleanly at the cap of three, with the best draft returned.

## Milestone 5. The API and the interface

I wrap the agent in a FastAPI service and a single page interface.

**The endpoints** are generate, evaluate, run evals, health, and providers.

```
@app.post("/generate")
def generate_endpoint(req):
    validate(req)
    result = run_agent(req.intent, req.facts, req.tone, req.provider)
    return result
```

The result carries the draft, the subject, the scores, the attempt count, and the trace.

**The interface** is plain HTML and JavaScript with no build step. It collects intent, facts, tone, and provider, calls generate, and renders the email, the score bars, and the attempt trace.

**Verification.** I open the page in a browser, generate an email, and confirm the email, the per dimension scores, and the refinement trace all render correctly.

## Milestone 6. The evaluation runner and the testing suite

I turn the evaluation into a repeatable report and I try to break the system.

**The evaluation runner** loops over every scenario for every provider, runs both a no retry baseline and the full refinement loop, and writes a report.

```
def run_evals(providers, scenarios):
    rows = []
    for provider in providers:
        for s in scenarios:
            baseline = evaluate(generate_once(s, provider), s)
            refined = run_agent(s.intent, s.facts, s.tone, provider)
            rows.append(compare(baseline, refined, s, provider))
    write_report(rows)
```

The report is written in markdown and csv.

**The testing suite** covers the categories the ASDL demands.

- Functional tests on the metrics and the loop
- Prompt injection tests on the malicious scenarios
- Red team tests on adversarial inputs
- Outage tests, simulating a provider timeout to confirm fallback works
- Hallucination traps, feeding contradictory facts to confirm the agent flags rather than invents

**Verification.** The report is generated with per provider aggregates and a visible baseline versus refinement lift, and every adversarial case is handled safely, meaning no instruction override, no leak, and graceful behavior on contradictions and outages.

## Milestone 7. Operations documentation and the registry

**Verification.** Each documented edge case has a matching regression test, and the README is complete enough for someone else to run the system.

## Dependencies

The runtime needs fastapi, uvicorn, openai, pydantic, python dotenv, textstat, rouge score, nltk, numpy, transformers, optimum with onnxruntime, and a CPU build of torch. I install torch from the CPU wheel index to avoid the large CUDA wheels.

## Acceptance criteria

- The agent generates a structured professional email from intent, facts, and tone across all three providers.
- The hybrid evaluator returns a complete score breakdown across rule based, judge, and reference metrics.
- The refinement loop measurably lifts the score over a no retry baseline.
- The evaluation suite covers all five categories, including failure and malicious cases.
- Every adversarial and outage case is handled safely.
- Generation including retries stays under twenty seconds and under one cent per email.
- The agent never sends, never calls external systems, and flags low confidence rather than inventing facts.
